# Лабораторная работа 15. Мультиплексирование

## Тема
Реализация TCP-сервера с использованием мультиплексирования ввода-вывода (`select`) и одного потока исполнения.

## Что делает программа
Сервер обслуживает несколько клиентских подключений в одном процессе без `fork()` и без потоков.

Поддерживаются команды:
- `CALC <op> <a> <b>` — вычисление арифметического выражения
- `FILE <filename> <size>` — приём файла от клиента
- `QUIT` — завершение соединения

Клиент подключается к серверу и предлагает два режима:
- вычисление
- отправка файла

## Идея решения
Вместо создания отдельного процесса для каждого клиента сервер использует `select()`.

Сервер хранит:
- слушающий сокет
- множество всех активных сокетов (`master`)
- копию множества для `select()` (`read_fds`)
- массив структур клиентов, где хранится состояние каждого подключения

Для каждого клиента есть два состояния:
- `STATE_WAIT_HEADER` — ожидание первой строки команды
- `STATE_RECV_FILE` — приём содержимого файла

Это позволяет серверу в одном цикле:
- принимать новых клиентов
- читать команды от уже подключённых клиентов
- принимать файлы по частям
- корректно отключать клиентов

## Почему это соответствует заданию
В задании требуется реализовать сервер из работы 13 с использованием мультиплексирования и одного потока исполнения.

В этой реализации:
- используется `select()`
- отсутствует `fork()`
- отсутствуют потоки
- несколько клиентов обслуживаются в одном процессе

## Состав файлов
- `server.c` — сервер
- `client.c` — клиент
- `Makefile` — сборка проекта

## Сборка
```bash
make
```

Или отдельно:
```bash
gcc -Wall -Wextra -Wpedantic -std=c11 server.c -o server
gcc -Wall -Wextra -Wpedantic -std=c11 client.c -o client
```

## Запуск
### Сервер
```bash
./server 12345
```

### Клиент
```bash
./client 127.0.0.1 12345
```

## Протокол обмена
### Калькулятор
Клиент отправляет строку вида:
```text
CALC + 5.000000 7.000000
```
Сервер отвечает, например:
```text
RESULT 12.000000
```

### Передача файла
Клиент сначала отправляет заголовок:
```text
FILE test.txt 29
```
затем отправляет `29` байт содержимого файла.

После успешного приёма сервер отвечает:
```text
OK FILE RECEIVED
```

## Проведённые тесты
### 1. Проверка сборки
```bash
make
```
Ожидаемый результат: без ошибок компиляции.

### 2. Проверка калькулятора
Ввод клиента:
```text
1
+
5
7
```
Ожидаемый ответ:
```text
Server response: RESULT 12.000000
```

### 3. Проверка деления на ноль
Ввод клиента:
```text
1
/
10
0
```
Ожидаемый ответ:
```text
Server response: ERROR division by zero
```

### 4. Проверка передачи файла
Создать файл:
```bash
echo "hello file" > test.txt
```
В клиенте выбрать:
```text
2
```
и указать путь:
```text
test.txt
```
Ожидаемый ответ:
```text
Server response: OK FILE RECEIVED
```

### 5. Проверка нескольких клиентов
Запустить сервер, затем открыть несколько терминалов с клиентами и выполнить запросы почти одновременно.
Сервер должен обслуживать их в одном процессе через `select()`.

## Важные замечания
1. Для `select()` используется ограничение `FD_SETSIZE`, что нормально для учебной лабораторной работы.
2. Имя файла не должно содержать `/`, `\\` и `..`.
3. Сервер закрывает соединение после обработки одного запроса, что соответствует логике данного клиента.

## Краткий вывод
Сервер из задания 13 был переделан с многопроцессной модели на мультиплексированную модель.
Теперь один процесс отслеживает все сокеты через `select()` и обслуживает нескольких клиентов без создания отдельных процессов.
