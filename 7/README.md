# Задание 07. Очереди сообщений POSIX

## Описание

В работе реализованы две программы для двухстороннего взаимодействия через POSIX очереди сообщений.

Программы обмениваются сообщениями по принципу **пинг-понг**:

- первая программа отправляет сообщение во вторую;
- вторая программа получает его и отправляет ответ обратно;
- обмен продолжается по очереди.

Для связи используются **две очереди сообщений**:

- `/q1` — сообщения от `prog1` к `prog2`;
- `/q2` — сообщения от `prog2` к `prog1`.

Для завершения обмена используется специальный приоритет сообщения:

- обычные сообщения — приоритет `1`;
- сообщение завершения — приоритет `10`.

Если одна из программ отправляет сообщение `exit` с приоритетом `10`, другая программа завершает работу.

## Структура проекта

- `prog1.c` — первая программа, создаёт очереди и начинает диалог;
- `prog2.c` — вторая программа, подключается к очередям и отвечает;
- `README.md` — описание проекта.

---

## Используемые функции POSIX

В программе используются следующие функции POSIX очередей сообщений:

- `mq_open()`
- `mq_send()`
- `mq_receive()`
- `mq_close()`
- `mq_unlink()`

---

## Почему для запуска используется Docker

На macOS отсутствует заголовочный файл `mqueue.h`, поэтому POSIX очереди сообщений в таком виде там не собираются напрямую.

Для решения задачи используется Docker с Linux-контейнером, внутри которого доступна поддержка POSIX message queues.

Таким образом:

- код остаётся написан на C под Linux;
- запуск выполняется на Mac через Docker;
- внутри контейнера можно нормально использовать `mqueue.h`.
