# Задание 08. Семафоры System V  
## Модель «производитель-потребители»

## Описание задания

В работе реализована модель «производитель-потребители» с использованием семафоров System V.

Процесс-производитель генерирует строки, состоящие из случайного количества случайных целых чисел, и записывает их в файл.

Процесс-потребитель анализирует файл: если в нём есть необработанные строки, потребитель берёт одну строку, находит в ней минимальное и максимальное число и выводит эти значения на экран.

Может быть запущено несколько экземпляров потребителя одновременно.

Доступ к файлу синхронизируется с помощью семафора System V.

Дополнительно предусмотрена передача имени файла в аргументах командной строки. Благодаря этому можно запускать несколько независимых экземпляров производителей и потребителей для разных файлов. Для каждого файла используется свой семафор.

---

## Идея решения

Общим ресурсом в задаче является файл.  
Чтобы несколько процессов не обращались к нему одновременно, для каждого файла используется отдельный семафор со значением `1`, который играет роль мьютекса:

- `1` — файл свободен
- `0` — файл занят

Производитель перед записью строки в файл захватывает семафор, после записи освобождает его.

Потребитель перед чтением файла также захватывает семафор, ищет первую необработанную строку, помечает её как обработанную, копирует числовую часть строки в память, закрывает файл и освобождает семафор. После этого уже вне критической секции вычисляет минимум и максимум.

Такой подход не позволяет нескольким потребителям обработать одну и ту же строку дважды.

Каждая строка файла начинается со служебного флага:

- `0` — строка ещё не обработана
- `1` — строка уже обработана
